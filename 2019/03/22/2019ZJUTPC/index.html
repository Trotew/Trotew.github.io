浙江工业大学第十七届大学生程序设计竞赛

Posted on 2019-03-22

这里是参赛须知。

比赛结束后题解也会同步在这里。

现场赛参赛须知

  1. 评测系统依然使用迎新赛的OJ。CE也算罚时，请务必小心。如果不清楚什么样会CE，可以提前到内网OJ上进行测试。链接
  2. 时限相较于标程都会放松至少两倍时限，绝大多数题目都会放超过$3$倍的时限。(比如标程$300ms$可以通过，那么时限就会放到$1000s$)，所以如果超时了就是真的超时。内存也会放大约两倍。
  3. 比赛应该不会提供电子版的cppreference,Javadoc，Python documentation等资料，如果有需要，请自行打印携带。
  4. 比赛严禁携带任何电子设备，任何电子设备都可能受到怀疑。比赛在按规定分配的电脑上进行，三个人一台电脑，三份题面。分配的电脑会切断外网。
  5. 比赛应该不会提供草稿纸和笔，请自备。（但是试题册可以当成草稿纸）
  6. 比赛可以在机房内讨论，但仅限于队内。严禁在厕所等位置交换思路和答案。
  7. G++版本为4.8，开启C++11和O2。至于具体什么能不能用请尽量提前做测试。

题目相关

  1. 题目册经过严格审查，如果有额外的题意不清等问题，可以举手向志愿者提问。
  2. 本场题目相较于区域赛较为简单，请集训队各参赛队伍轻虐。但是对于很少参加类似竞赛的同学来说可能具有相当高的难度，希望心态放平，稳定求胜。
  3. 命题组假定所有参赛选手都了解常见算法，能编写简单的小程序，对C++的基本操作有清晰的认识，故在题目中如果需要使用到一些不常见的技巧将不做提醒。
  4. 命题组假定所有参赛选手都了解算法竞赛等程序设计竞赛的基本提交方式，在此不再赘述。
  5. 所有标程都使用C++撰写，其他语言不保证时限和内存的放宽。
  6. 所有标程默认都使用scanf()读入，如果执意使用cin在读入上会有TLE的风险，因为这种原因可能造成的TLE，在试题册上将不再提醒。
  7. 命题范围与ICPC竞赛一致。
  8. 某些试题如果说明是单组数据，是为了方便写程序。提交的程序会对不同的数据分别独立运行。请尽量不要尝试对所有的数据都直接输出相同的答案，因为这样的程序不可能AC。
