E. 简单结论+暴力

题意

有$n$只队伍，每个队伍有一个封榜前的得分$a_i$，有一个封榜后的得分$d_i$，得分越大越靠前，如果有多支队伍得分相同，队伍$id$小的靠前。

你可以决定展示封榜后结果的队伍顺序，如果展示前一支队伍的排名为$x_i$，展示后这支队伍的排名为$y_i$，那么观众将鼓掌$|x_i-y_i|$的时间。

问最大的鼓掌时间。

$n,a_i\le 100$

$-100\le d_i\le 100$

题解

以下两个结论请尝试认真思考并加以证明。

引理1：两支队伍$u,v$，当且仅当如果先展示$v$，则$u$无法超越$v$，而如果先展示$u$，则$u$可以超越$v$；再展示$v$，$v$又可以超越$u$时。$u$必须在$v$之前展示。其余时候谁先谁后并不影响答案。

引理2：上述方式产生的$u\Rightarrow v$连有向边后，图必然是一张有向图。

由上，暴力连边后拓扑排序求答案即可。

时间复杂度$O(n^3)$

F. 动态规划

题意

一开始你有$b$个超级硬币。接下来你要吃$n$天的饭，每天会花费$a_i$元，对于每一天，你可以通过这种方式利用超级硬币：

假设当天需要花费$r$元钱，你选择使用$x$个超级硬币。($x\le \min(b,r/2)$)，当天可以少付$x$元钱。并得到$\lfloor(r-x)/10\rfloor$个超级硬币。

问最小的总付款钱数和方案。

$n\le 5000,\ b\le 1e5$

题解

题目看似贪心，但是深思以后觉得贪心相当不适，决定动态规划，但是方程也很不好找，所以放弃了。

我们需要一种状态转移方程，使得能记录或快速找到当前拥有的超级硬币数量和当前的最小付款即可。

经过一番思考（然后失败了并看了题解，这里省略数次失败的心路历程）,我们使用如下的状态定义：

$d[i][j]$表示第$i$天结束时，额外得到了$j$个硬币的最小付钱数量。

此时可以得到对于某个状态，当前的超级硬币数量为$b+j$，转移起来极其舒适了。

显然有状态转移方程：

$d[i][j]=\min{ d[i-1][k] + \cdots }$，emmm好像列不出来

不过问题不大，我们对于每种状态，枚举每天花了几个硬币，这总好了，适当转移一下。

时间复杂度$O(c*k^2)$，其中$c$约为常数$\frac{1}{40},\ k=\sum a_i$

只要胆子大，1e8不怕

I. 贪心

题意

现在有$n$件物品，每件物品可以放进$A$中获得$a_i$，也可以放进$B$中获得$b_i$。

现在$A$中可以容纳$a$件物品，$B$中可以容纳$b$件物品，问放进$A,B$中的物品的获得利益最大是多少，并输出方案。

$n,a,b\le 3000$

题解

结论：可以先贪心装满$A$，然后考虑装满$B$。

  • 选出一件没有装入$A$的物品$i$，并且装入$B$，获得$b_i$。
  • 选出一件已经装入$A$的物品$j$，并且装入$B$，并选出一件没有装入$A$的物品$i$装入$A$，获得$b_j-a_j+a_i$。

两边取最大即可。

上述结论可以利用网络流证明（本质是优化网络流，直接跑网络流也可以AC）。

时间复杂度$O(n\log n)$

当然如果想DP也可以，但是很麻烦。

K. 构造图论

题意

给定$n$个点$m$条边的一张无向图，现在需要你将所有边定向，使得无论怎么走都能从$s$点走到$t$点。

还需要满足一些条件：

  • 有向图中不能有环
  • $s$必须是唯一一个入度为$0$的节点
  • $t$必须是唯一一个出度为$0$的节点。

如果有办法的话，输出整张图的定向方案。

$n\le 4e5,\ m\le 1e6$

题解

思考了很多不可行方案。

正解是在dfs树上做炫酷操作（根本搞不懂）。
