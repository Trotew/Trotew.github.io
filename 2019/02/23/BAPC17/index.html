B. 几何

题意

平面上给定$n$个整数点圆，给定起点和终点，问在圆上或圆内行走的最短距离。

$n \le 25$

坐标范围$\le 1e6$

题解

可以想到最短路径一定由一些线段组成，且端点为圆和圆间的交点或起点终点。

两个圆之间最多只有 2 个交点， 所以可能连成线段的点最多有 $n^2 + 2$ 个。

可以暴力枚举每两个点，判断其连成的线段是否完全被圆覆盖。

把所有边连好后做一遍最短路。

难点在于如何判断线段被圆覆盖。

一种方法是，用当前线段和所有圆做交，把得到的交点按坐标排序，做扫描线判断线段是否被完全覆盖。

这种方法不一定是最优解，因为精度被卡得很死，代码量也比较大。

算法总体复杂度为 $O(n^3)$。

E. 二分答案+二分图匹配

题意

平面上给定b个白点和r个黑点，选取其中n个点，使得异色最近距离最大。

$b,r,n \le 250, n\ge b+r$

坐标范围绝对值 $\le 10000$

题解

显然可以二分答案。

问题转化为在限定异色最近距离为$r$的前提下，最多能选取多少个点。

注意到如果选取了一个白点，与其最近距离小于$r$的异色点都不能选取，这个流程疑似二分图匹配中增广路的操作。

稍加思考后，发现问题可以使用二分图匹配判断，把所有相互冲突的点连边，答案即为$b+r-k$，其中$k$为二分图匹配的答案。

G. 状压DP

题意

$m$ 个人之间存在 $n$ 条欠钱关系（如 $a$ 欠 $b$ $w$块钱）。

求相互之间至少需要还几次钱，使得所有人都不欠钱（所有人身上的钱是无限的）。

$1 \le m \le 20$

$1 \le n \le 1000$

题解

$S$ 表示人的集合，如果 $S$ 中的人相互还钱，可以使得收支平衡，称 $S$ 为一个平衡集。

如果 $S, T$ 都是平衡集，且 $T \subsetneq S$，那么 $S - T$ 必然也是平衡集。

$f(S)$ 表示 $S$ 中最多有几个平衡集。

转移方程：$f(S) = \max\limits_{i \in S} f(S - { i }) + balance(S)$。

答案为 $m - f({ 1, 2, \ldots , m })$。

时间复杂度 $O(m \cdot 2^m)$。

H. 几何

题意

给定$n$条线段，问形成了多少个不同的闭合图形。

$n\le 1000$

坐标范围$\le 1e9$

题解

先随便画几个图案找一下规律。

可以发现如果连接了$k$条这$k$条间都互相连接的线段，产生的新的闭合图形数量为$k-1$。

另外注意到如果出现三线或者多线相交，需要减去多余部分。

以上，使用线段交的几何模板即可。

时间复杂度$O(n^2)$

J. 数据结构+思维找规律

题意

数轴上，一开始有$n$只青蛙，第$i$只的位置为$a_i$，每只青蛙第$i$步跳距离为$i$。

$q$次询问，每次可以选择：

  • 加入一只位置为$p_i$的青蛙。
  • 移除一只位置为$p_i$的青蛙。
  • 将他们的目的地改为$p_i$。

对于每次询问，输出所有青蛙到达目的地的总最小步数。

$n\le 5000,q\le 1e6,$坐标范围$\le 1e6$

数据保证全场加减青蛙次数不超过5000次。

题解

数据提示暗示我们快速查询操作$3$，缓慢查询操作$1$和$2$。

又注意到坐标范围非常有限，想象如果能够足够快地更新所有区间，并且快速单点查询就好了。

我们研究一下青蛙如果从位置$0$开始，往右跳，一些位置的最短步数。

通过一个简单的dp，我们可以算出一些位置的简单步数。

注意到先是$1$，随后$3$，随后$2,4,6$，随后$8,10$，随后$5,7,9,11,13,15$。

注意到奇偶交替的规律，向左扫也同样。

为了实现奇偶分开染色，建立两棵线段树分别查询即可。

时间复杂度$O(n\sqrt k\log k+q \log k)$，其中$k$为坐标范围。

K. 图论

题意

有$n$个人，上擂台斗殴，对于任意两两间的斗殴，都必有一方一定会胜利，且永远不会改变。

请你安排一种上擂台的顺序使得$1$会成为最后的赢家。

如果不可能，输出”impossible”。

$n\le 1000$

题解

考虑建立一张有向图，如果$i$能打败$j$则连边$(i,j)$。

那么以1为起点，如果能到达所有的其他点，则一定可以获胜。

考虑任意顺序dfs，并标记访问标记，构建dfs树。

对于任意一个非叶子节点$a$，

  • 如果其仅有一条连出去的边$(a,b)$，那么显然先让$b$（及其孩子）上擂台，再让$a$上，最终$b$会被$a$打败。
  • 如果其有多条边$(a,b),(a,c),\cdots$，那么先解决其中一个子树的问题即可，最终无论是$b$打败$c$或$c$打败$b$都不影响被$a$打败。
